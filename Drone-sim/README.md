# PROJECT MONERO:  DRONE SIMULATION

## Protocol: Narrative Mode (The Perimeter Breach)

In a near-future landscape defined by the digital divide, the Monero-class drones are the only operational ghosts capable of traversing the restricted "Gutter" zones. These automated sentinels are designed for high-stakes retrieval and surveillance in environments where traditional communication is jammed and visibility is a luxury.

As a Pilot Link operator, you are the final point of failure. Your mission is to maintain the integrity of the Monero fleet, navigate the anomalous terrain signatures of the restricted sector, and ensure that every unit returns to the Solar Pad for high-velocity energy restoration. The terrain is treacherous, defined by shifting data-mountains and deep valley voids. The perimeter is absolute; crossing the 285-meter geofence will trigger a hard-reset of the neural link. 

Your craft is a specialized Antigravity unit, capable of instantaneous multi-axis movement and equipped with the proprietary Phoenix-class Supercharging system. Every flight is a dance with entropy. Watch your energy levels, master the manual override, and secure the perimeter.

## Technical Specifications (v38.0 Deployment)

### 1. Freedom Flight & Smart Docking
The Monero system provides total operational freedom. Operators can manually land at the central Solar Pad at any time. The Smart Docking engine detects contact within 8.8 meters and initiates a high-velocity 920 units/sec charge link, restoring full power in under 10 seconds.

### 2. Tactical Customization
The Monero fleet supports six tactical hull color-ways for operational diversity:
- **Cyan**: Standard Neural Link.
- **Phoenix**: High-Heat Restricted Access.
- **Toxic**: Bio-Hazardous Zone Mapping.
- **Cobalt**: Deep-Space Maritime Surveillance.
- **Amethyst**: Low-Light Urban Infiltration.
- **Ghost**: Stealth Calibration Mode.

### 3. Ultra-Fidelity Navigation
Powered by a high-intensity directional lighting array (4.5x output), the simulation renders terrain with surgical detail. Every peak and valley is rendered to ensure the pilot has maximum depth perception during high-speed manual maneuvers.

### 4. Pilot Override Mastery
The simulation supports high-precision input via Keyboard and Gamepad API:
- **WASD**: Primary Axis Movement.
- **Q/E**: Yaw (Turning Mastery).
- **Space/Shift**: Altitude Correction.
- **Mission Link**: Automated Python/Blockly logic for precise mission execution.

## Deployment Instructions

### Prerequisites
- Node.js (Latest stable version)
- npm (Node Package Manager)

### Local Implementation
1. Clone the repository to your workstation.
2. Initialize dependencies:
   ```bash
   npm install
   ```
3. Boot the neural link (Launch dev server):
   ```bash
   npm run dev
   ```
4. Access the cockpit via `http://localhost:5173`.

## System Safety Protocols
- **Sequential RTB**: If energy levels drop below critical thresholds (15%), the drone will automatically activate the Return-to-Base sequence.
- **Geofence Boundary**: A hard-stop logic is active at 285 meters to prevent signal loss beyond the simulation theater.
- **Pilot Link Visuals**: Manual Override mode triggers a screen-space glow to confirm direct human control.
